# Attack Path Management

Attack Path Management (APM) is a proactive, continuous security process focused on identifying, mapping, and closing the routes an attacker could take to reach an organisation's critical assets.

Unlike traditional vulnerability management - which looks for unpatched software or misconfigurations - APM looks at identity-based risks. It focuses on the chain of legitimate permissions that can be abused.

## Key Components of APM

* **Continuous Mapping:** Regularly updating the graph of relationships as the network changes.
* **Choke Point Identification:** Finding the one or two bottleneck permissions that, if removed, would eliminate hundreds of potential attack paths simultaneously.
* **Blast Radius Analysis:** Calculating exactly how much damage a single compromised workstation could cause based on its current permissions and active sessions.
