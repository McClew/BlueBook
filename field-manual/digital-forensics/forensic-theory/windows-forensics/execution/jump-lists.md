# Jump Lists

Jump Lists provide a detailed mechanism for tracking user interaction with files and applications directly from the Windows taskbar. They split data across two distinct system directories.

Jump Lists are broken down into two main formats based on whether the OS or the application itself logged the activity:

*   **Automatic Destinations:**\
    Generated by the Windows shell when files are opened.

    Path: `C:\Users\<User>\AppData\Roaming\Microsoft\Windows\Recent\AutomaticDestinations`
*   **Custom Destinations:**\
    Generated by specific application developers to pinpoint pinning behaviors or unique application tasks.

    Path: `C:\Users\<User>\AppData\Roaming\Microsoft\Windows\Recent\CustomDestinations`

Both variations utilise the MS-CFB (Microsoft Compound File Binary) layout, functionally acting as a miniature file system within a single file. Within an `AutomaticDestinations` entry, the OS packs individual streams that mirror the exact structure of standard binary Link (LNK) files.

Parsing a single Jump List entry can reveal launch counts, internal creation dates, and full target directory structures, remaining intact even if the primary LNK file in the `Recent` folder has been wiped.
