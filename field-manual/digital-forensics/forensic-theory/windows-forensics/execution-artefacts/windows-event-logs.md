# Windows Event Logs

Windows Event Logs provide definitive, cryptographic-grade, kernel-validated tracking of process execution, moving completely away from the easily manipulated shell-level logs.

<figure><img src="../../../../../.gitbook/assets/image (25).png" alt=""><figcaption><p>Event 4688 Detailed Process Creation Event.</p></figcaption></figure>

Event logs are written as binary XML (`.evtx`) streams. Process execution logging requires specific audit policies or additional system drivers to be explicitly enabled.

**Path:** `C:\Windows\System32\Winevt\Logs\`

***

## Event IDs

#### Security.evtx - Event ID 4688 (Process Creation)

Captures the exact binary launched, token elevation levels (such as administrative vs. standard user context), and the Creator Process Name (the parent application that called the launch).

{% hint style="info" %}
#### Improved Logging

Event ID 4688 can be configured via Group Policy to log the _Process Command Line_ string. This is invaluable when hunting for living-off-the-land attacks (such as `vssadmin.exe delete shadows /all`).

Furthermore, Sysmon Event ID 1 tracks the ProcessGuid, a unique identifier generated by the system. This allows you to map out an unbroken process tree execution timeline across the host, completely defeating malware that attempts to hide by quickly spawning and killing process child layers.
{% endhint %}

#### Microsoft-Windows-Sysmon/Operational.evtx - Event ID 1 (Process Creation)

An advanced telemetry log that captures command-line arguments and automatically records file hashes (SHA-256/MD5) of the running application on the fly.

#### Microsoft-Windows-PowerShell/Operational.evtx - Event ID 4104 (Script Block Logging)

Captures the complete, raw content of executed PowerShell code strings.
