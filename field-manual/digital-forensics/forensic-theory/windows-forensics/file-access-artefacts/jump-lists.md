# Jump Lists

The taskbar interaction framework providing application-specific document telemetry histories, capturing application IDs (AppIDs), launch counters, destination paths, and granular lifecycle timestamps.

Jump Lists are broken down into two main formats based on whether the OS or the application itself logged the activity:

*   **Automatic Destinations:**\
    Generated by the Windows shell when files are opened.

    Path: `C:\Users\<User>\AppData\Roaming\Microsoft\Windows\Recent\AutomaticDestinations`
*   **Custom Destinations:**\
    Generated by specific application developers to pinpoint pinning behaviors or unique application tasks.

    Path: `C:\Users\<User>\AppData\Roaming\Microsoft\Windows\Recent\CustomDestinations`

{% hint style="info" %}
#### Evidence Resilience

Because `AutomaticDestinations` utilise the Compound File Binary format to store internal LNK streams, they retain independent copies of target file metadata. Even if a suspect uses CCleaner or similar script utilities to wipe the standard `Recent` shortcuts folder, the underlying Jump List database streams often escape cleanup routines due to structural locks, preserving valuable timeline data.
{% endhint %}
